# 动态流式二维码 v2.3

> 通过屏幕闪烁的二维码，将文本和文件从电脑传输到手机——无需网络、无需数据线。

## 简介

动态流式二维码是一个纯离线的点对点数据传输工具。发送端在电脑屏幕上快速轮播一系列二维码，接收端用手机摄像头逐帧扫描，收集并还原数据。整个过程不依赖任何网络连接，仅通过光信号完成传输。

## 特性

- **文本传输** — 粘贴任意长度文本，自动压缩、分帧、轮播
- **文件传输** — 支持任意格式文件（PDF、图片、压缩包等）
- **数据完整性** — SHA-256 校验，确保传输无损
- **高效压缩** — gzip 压缩，大幅减少传输量
- **紧凑协议** — 13 字节二进制头，最大化有效载荷
- **预渲染帧** — 离屏 Canvas 预渲染，播放流畅不卡顿
- **传输摘要** — 完成后显示帧数、用时、传输速率

## 快速开始

### 发送端（电脑浏览器）

1. 打开 `send/index.html`
2. 输入文本，或点击「选择文件」上传文件
3. 点击「开始发送」
4. 将屏幕亮度调到最高

### 接收端（手机浏览器）

1. 打开 `recv/index.html`
2. 允许摄像头权限
3. 将手机摄像头对准电脑屏幕上的二维码
4. 等待扫描完成，自动显示文本或提供文件下载

## 技术架构

```
发送端                                    接收端
┌──────────────┐                    ┌──────────────┐
│  文本/文件    │                    │   摄像头      │
│      ↓       │                    │      ↓       │
│  gzip 压缩   │                    │   jsQR 解码   │
│      ↓       │                    │      ↓       │
│  SHA-256 哈希 │    QR 二维码       │  解析 v3 头   │
│      ↓       │  ═════════════►    │      ↓       │
│  分帧 + 编号  │    屏幕 → 摄像头    │  收集帧       │
│      ↓       │                    │      ↓       │
│  生成 QR 码   │                    │  SHA-256 校验  │
│      ↓       │                    │      ↓       │
│  循环轮播     │                    │  gzip 解压    │
└──────────────┘                    └──────────────┘
```

## 协议格式

### v3 二进制头（13 字节）

| 偏移 | 长度 | 说明 |
|------|------|------|
| 0-1 | 2 | 魔数 `v3`（0x76 0x33） |
| 2 | 1 | 标志位：bit0=已压缩, bit6-7=类型(01=数据) |
| 3-4 | 2 | 帧号（小端 uint16） |
| 5-6 | 2 | 总帧数（小端 uint16） |
| 7-10 | 4 | 原始数据长度（小端 uint32） |
| 11 | 1 | 保留 |
| 12 | 1 | 数据帧数 k |

### 数据帧结构

```
[v3头 13字节] + [payload 字节]
```

### payload 结构（发送端构建）

```
[SHA-256 32字节] + [gzip压缩数据(含填充)]
```

### 文件传输格式

压缩前的数据结构：

```
[文件名长度 1字节] + [文件名 UTF-8] + [文件内容]
```

文本传输直接压缩 UTF-8 编码的文本字节。

## 使用建议

- **屏幕亮度**调到最高
- **手机距离**屏幕 20-40cm
- **保持稳定**，避免晃动
- **环境光线**不要太暗
- 传输大文件时选择**更高帧率**（15/20 FPS）
